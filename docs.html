AURA Commerce Engine

Drop-in e-commerce for any website. Use complete account dashboards or build custom with attributes — zero CSS required, zero HTML prescribed.

💡 How It Works

The engine works through HTML attributes. You don't need specific elements — any element works.

Just add the data-commerce attribute to any element on your page. The engine detects it and adds the functionality. You choose the HTML, you choose the styling. The engine only cares about the attribute.

✅
Element-agnostic. Use a <div>, <button>, <a>, <span>, or anything else. The attribute works the same regardless of which HTML element you use.

Think of each data-commerce value as a function. Adding it to any element enables that function on that element. Like shortcode parameters — add what you need, skip what you don't.

🛍 Product Listing

Displays a grid of products. Add filter options to control what shows.

data-commerce="product-list"
Container for a product grid. The engine fills it with products from your store. Place a product-template inside it.
data-sort="price-low"Sort: price-low / price-high / newest / name
💡
Note: Detailed filtering attributes and sidebar controls are now grouped in the Smart Filtering section below.
🔀
Combine freely: data-commerce="product-list" data-type="featured" data-category="sneakers" data-limit="3" → 3 featured sneakers.
data-commerce="product-template"
Place inside a product-list. This element gets cloned for each product. Use data-field attributes inside to fill product data. Use <template> tag or any hidden element.

How the Template System Works

The template is the heart of product rendering. Here's exactly what happens step by step:

StepWhat the Engine Does
1. DiscoverEngine scans the page for any element with data-commerce="product-list"
2. Read FiltersReads data-type, data-category, data-limit from the container to build the API query
3. FetchCalls the API: GET /api/v1/public/products?category=outerwear&featured=true&limit=4
4. Find TemplateLooks for a child element with data-commerce="product-template" inside the container
5. CloneFor each product returned by the API, clones (duplicates) the template element
6. Fill FieldsInside each clone, finds all data-field elements and fills them with the product's data
7. Set IDSets data-product-id on the clone so cart/wishlist buttons know which product they belong to
8. BindBinds click handlers on any add-to-cart or wishlist-toggle elements inside the clone
9. AppendAdds the filled clone to the container. The original template stays hidden.
💡
Your template = your design. The engine never creates HTML structure. It only fills data-field elements that YOU already placed inside your template. Whatever you design is what gets repeated for each product.

Advanced: Mustache Syntax ({{ }})

In addition to data-field, Engine v3 supports Mustache-style placeholders inside templates. This is mandatory for attributes like href or src where you need dynamic IDs or URL fragments.

PlaceholderEffect
{{title}}Renders product title
{{price}}Renders product price (numeric)
{{currency}}Renders store currency symbol
{{imageUrl}}Renders product main image URL
{{slug}}Renders product URL slug
{{id}}Renders product unique database ID
{{description}}Renders raw HTML description
🚀
Pro Tip: Use {{slug}} inside links: <a href="product.html?slug={{slug}}">. Standard data-field cannot modify partial URL attributes, but placeholders can!

Zero Code Sync: No JavaScript required. As long as your variants have images in the backend, the engine handles the swap across all collection pages.

OptionDefaultEffect
data-skindefaultWishlist design skin. Future skins: 2, 3, 4

Custom Wishlist Layout

If you don't want the default grid, you can design your own card inside the container. The engine will use it as a template and clone it for every product in the wishlist.

  1. Place a single div with data-commerce="wishlist-item" inside your wishlist page container.
  2. Inside it, design your card and tag the elements:
    • data-commerce="wishlist-item-image" on an `img`
    • data-commerce="wishlist-item-name" on a text element
    • data-commerce="wishlist-item-price" on a text element
    • data-commerce="wishlist-item-link" on the parent `a` tag (if you want it to wrap the image/title)
    • data-commerce="wishlist-toggle" on your remove button

The engine will automatically grab your design, clear it, and regenerate the list using your custom styling!

💡
Custom APIs: If you want to build a completely custom design from scratch, you can still fetch wishlist data via JS: `engine.api('/customer/wishlist')` (Logged In) or `engine.api('/public/products?ids=' + engine.getWishlist().join(','))` (Guest).

👤 Authentication

Login, register, logout, and conditional visibility.

Fast Auth (Modal & Redirect)

Open a login/register popup or redirect to a custom page instantly. Note: These hide automatically when the user is already logged in.

data-commerce="customer-login"
Clicking opens the default login/register modal.
data-commerce="customer-login-page"
Clicking redirects to a custom login page. Usage: <a href="/login" data-commerce="customer-login-page">. It automatically adds a ?redirect= param so the user returns to the current page after login.

Custom Auth Pages (Inline Forms)

If you want to design your own "Login" or "Register" pages, use these plug-and-play forms. They generate complete, functional forms automatically.

data-commerce="login-form"
Renders a complete login form with email, password, and forgot password link.
data-commerce="register-form"
Renders a registration form with full name, email, and password.
data-commerce="forgot-password-form"
Renders a "Forgot Password" form to send reset links.
OptionDefaultEffect
data-skindefaultForm design skin.
data-redirect/my-accountPage to redirect to after successful login/registration.

Control Visibility

Show or hide elements based on user state:

data-commerce="account-link"
Visible only when logged in. Use with an href to link to the account page. Note: Hides automatically when logged out.
data-commerce="customer-logout"
Clicking logs out the customer and resets the UI. Note: Hides automatically when logged out.
💡
Smart Buttons: customer-login, customer-login-page, account-link, and customer-logout all have built-in "Smart Visibility". You don't need to wrap them in guest-only or auth-only blocks — the engine handles the logic for you!

Manual Visibility Control

For custom sections (like a "Welcome Guest" banner), use these attributes:

data-commerce="guest-only"
This element is visible only when NOT logged in. Hidden as soon as a customer is authenticated.
data-commerce="auth-only"
This element is visible only when logged in. Hidden when no customer is authenticated.

🔍 Search

Instant product search with dropdown results.

data-commerce="search"
Put on any container element. Engine auto-generates a search input and results dropdown inside it. Minimum 2 characters to trigger search. Clicking a result navigates to the product page.
💡
If you already have an <input> inside the container, the engine uses yours instead of creating one. This lets you control the input's styling.
